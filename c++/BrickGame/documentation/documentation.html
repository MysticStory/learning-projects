BrickGame - Описание

Общее описание

BrickGame - это набор двух игр: Змейка и Тетрис. Каждая игра имеет свои правила и управление. Ниже приведены подробные описания и правила для каждой игры, а также конечные состояния игры.

Змейка

Описание

В игре Змейка цель состоит в том, чтобы управлять змейкой, которая растет по мере того, как она ест еду. Игрок должен избегать столкновений с границами экрана и собственным телом.

Правила

  • Используйте клавиши со стрелками для управления движением змейки.
  • Ешьте еду, чтобы змейка росла.
  • Избегайте столкновений со стенами и собой.

Конечные состояния

Змейка имеет следующие конечные состояния:

  • MainMenu - Главное меню
  • Start - Начало игры
  • Paused - Игра приостановлена
  • SpawnFruit - Появление еды
  • TurnMove - Движение змейки
  • EatFruit - Употребление еды
  • GameOver - Игра окончена
  • Win - Победа

Тетрис

Описание

В игре Тетрис цель состоит в том, чтобы размещать падающие блоки, чтобы заполнить горизонтальные линии. Заполненные линии удаляются, и игрок продолжает до тех пор, пока блоки не достигнут верха экрана.

Правила

  • Используйте клавиши со стрелками для управления блоками.
  • Заполняйте горизонтальные линии, чтобы они исчезли.
  • Не допускайте, чтобы блоки достигли верха экрана.

Конечные состояния

Тетрис имеет следующие конечные состояния:

  • MainMenu - Главное меню
  • Start - Начало игры
  • Paused - Игра приостановлена
  • SpawnShape - Появление формы
  • Moving - Движение формы
  • Shifting - Изменение формы
  • Attaching - Прикрепление формы
  • GameOver - Игра окончена
  • Win - Победа
BrickGame - Документация

BrickGame - Документация

Общее описание

BrickGame - это программная реализация двух игр: Змейка и Тетрис. Каждая игра имеет свои состояния, классы и методы для управления игровой логикой. Данная документация описывает структуру проекта, классы, состояния и методы, используемые в игре.

Архитектура

Проект состоит из двух основных компонентов: игры Змейка и игры Тетрис. Каждая из них реализована с использованием паттерна состояния, где каждое состояние игры представлено отдельным классом. Эти классы управляют логикой игры в зависимости от текущего состояния.

Классы и состояния

Змейка

Класс SnakeModel

Класс SnakeModel отвечает за управление состояниями игры "Змейка". В нем реализованы основные методы для перехода между состояниями и выполнения логики игры.

Методы:
  • void setState(GameState state) - Устанавливает текущее состояние игры.
  • void resetGame() - Сбрасывает параметры игры к начальному состоянию.
  • void spawnSnake() - Создает новую змейку на поле.
  • void updateGame() - Обновляет текущее состояние игры.

Состояния

Игра Змейка имеет следующие состояния, каждое из которых представлено отдельным классом:

  • MainMenu: Главное меню
  • Start: Начало игры
  • Paused: Игра приостановлена
  • SpawnFruit: Появление еды
  • TurnMove: Движение змейки
  • EatFruit: Употребление еды
  • GameOver: Игра окончена
  • Win: Победа

Тетрис

Класс TetrisModel

Класс TetrisModel отвечает за управление состояниями игры "Тетрис". Он контролирует логику игры, такие как появление новых фигур, их перемещение и обработку столкновений.

Методы:
  • void setState(GameState state) - Устанавливает текущее состояние игры.
  • void spawnShape() - Создает новую фигуру на поле.
  • void moveShape() - Перемещает текущую фигуру.
  • void updateGame() - Обновляет текущее состояние игры.

Состояния

Игра Тетрис имеет следующие состояния, каждое из которых представлено отдельным классом:

  • MainMenu: Главное меню
  • Start: Начало игры
  • Paused: Игра приостановлена
  • SpawnShape: Появление фигуры
  • Moving: Движение фигуры
  • Shifting: Изменение формы фигуры
  • Attaching: Прикрепление фигуры
  • GameOver: Игра окончена
  • Win: Победа
